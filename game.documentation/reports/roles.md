# Roles
This document explains the different responsibilities of each role

## Game Director 

Responsibilities:
- Overall vision and design cohesion
- Scope management and feature prioritization
- team coordination
- Final decision authority on mechanics and endings
- Does research into topic to keep the game realistic and ensuring it matches with course goals
- Helps out in different areas where ever help is necesarry 
- ensures end result checks all the rubric requirements

## Systems Designer

Responsibilities:

- Define metrics and how they interact (engagement, revenue, user mental status, public trust, investor confidence, etc)
- Define how metrics change when certain desisions are made
- Set thresholds for each ending
- Ensure balanced gameplay, each ending has a realistic likelyhood. 
- Works closely with Narrative designer to match decisions made with outcomes.

## Narrative Designer

Responsibilities:

- Create dialogue, events, and stakeholder interactions
- Write branching storylines and endings
- Tie system outcomes to narrative consequences
- Give characters unique personalities and reactions
- Works closely with system designer to match decisions made with outcomes

## Lead Programmer 

Responsibilities:

- Backend Design (metrics update, decision processing)
- Implement event triggers and stakeholder reactions
- Turn narritave into code, (if this decision is made, then that happens) 
- works closely with UI/UX designer to link frontend to backend

## UI/UX Designer

Responsibilities:

- Frontend Design (dashboards, decision interfaces, and visual feedback)
- Make sure that complex metrics are intuitive to understand
- works closely with Lead programmer to link frontend to backend
